# 사주(四柱) 계산 근거

## 앱에서 쓰는 방식

- **四柱(년·월·일·시 주)** 는 [`lunar-javascript`](https://github.com/6tail/lunar-javascript)의 **八字(EightChar)** 로 계산합니다.
- **절기·입춘** 을 반영한 월주·연주를 쓰므로, 연도별 절기 시각표를 직접 유지할 필요가 없습니다.

## 이전 이슈 (로컬 `saju-db.js` 단독 계산)

- `lib/saju-db.js` 안 **절기 데이터**는 샘플로 **2020년·2026년만** 있었고, 그 외 연도는 `get절입월`이 **양력 월을 그대로 반환**해 **월주가 틀어질 수 있었습니다**.
- 연주도 **입춘 전후**를 구분하지 않는 단순 공식만 있으면 오류가 납니다.

→ UI용 사주는 **`lib/saju-lunar.ts` → `calculateSajuFromSolar`** 경로만 사용합니다.  
→ `saju-db.js`의 **천간·지지·오행 해석·퍼센트 집계**는 그대로 활용합니다.

## LLM으로 사주를 돌릴까?

- **팔자 간지 자체**는 규칙·절기가 정해져 있어 **엔진(라이브러리)이 맞고**, LLM은 **해석 문장**에 쓰는 편이 안전합니다.
- LLM에 생년월일만 주고 간지를 “추측”하게 하면 **환각·불일치** 위험이 큽니다.

## 외부 API

- `app/api/saju/route.ts` 는 (키가 있을 때) 별도 **FREEASTRO** 바이저 API를 호출합니다. 메인 페이지의 `runCalculations`와는 **다른 경로**입니다.
